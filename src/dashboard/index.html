Start commands

Run from the project root (folder that contains src/ and config/). On Windows, use py instead of python if needed.

Web UI: http://127.0.0.1:8081 — port is config/settings.yaml → dashboard.dashboard_port.

Paper trading + dashboard + AI

Default run: simulated orders (not --live). LLMs are off unless you set ai.enabled: true and a non-empty provider_chain in config/settings.yaml, with keys in config/secrets.env. Use per-strategy use_ai in YAML to control which strategies call the model.

python start.py

Equivalent: python src/main.py --paper (see start.py at repo root).

Bot (paper / default dry-run)

python src/main.py
python src/main.py --paper

Dashboard only (no trading loop)

python src/main.py --dashboard-only

Live trading

Requires typing YES at the prompt. Use only when you intend real orders.

python src/main.py --live --confirm-live

Kill switch & resume

python src/main.py --emergency-stop
python src/main.py --resume-trading

Backtest (CLI)

For local use or a separate scheduled job. On Railway (or any single-process deploy), keep the service on --paper / --live and run backtests from the Backtest tab instead — main.py --backtest as the container CMD replaces the long-lived bot.

Entry point forwards to scripts/run_backtest.py; extra args go after --backtest.

python src/main.py --backtest --plan config/backtest_plan.yaml
python scripts/run_backtest.py --help

Unit tests

python -m pytest tests/ -q
